# BTCUSDT Dashboard v5 系統改良與升級報告

**作者**：Manus AI
**日期**：2026年4月18日

## 升級背景與目標

在完成 BTCUSDT Dashboard v5 的基礎部署後，為了進一步提升系統的可觀測性與策略治理能力，我們根據現有的改進報告進行了一系列深度改良。本次升級的核心目標在於提供更直觀的策略診斷資訊，並透過自動化分析輔助交易者進行參數調整。同時，我們也對前端介面的文案進行了全面審核與修正，確保使用者體驗的一致性。

## 核心改良項目

本次系統升級涵蓋了後端引擎擴充與前端介面重構，主要分為以下四個核心模組：

### 1. 家族聚合診斷引擎

在原有的單一策略診斷基礎上，我們引入了「策略家族」的概念。後端 `diagnostics_engine.ts` 現在會將五個運行中的策略歸類為五大核心家族：**PA 價格行為**、**趨勢回踩**、**結構確認**、**趨勢確認**與**均值回歸**。

透過計算各家族整體的發送率、活躍率與阻擋率，交易者可以快速判斷目前市場環境對哪一種類型的策略最為友善。例如，當趨勢回踩家族的活躍率顯著高於均值回歸家族時，即暗示市場正處於明確的單邊行情中。

| 策略家族 | 包含策略 | 觀察重點 |
|---------|---------|---------|
| PA 價格行為 | PA 主力 | 價格動能與關鍵支撐阻力 |
| 趨勢回踩 | HWR-B | 趨勢延續性與回調深度 |
| 結構確認 | CannonBall | 市場結構轉換與流動性 |
| 趨勢確認 | EMA Cross | 中長線趨勢共振 |
| 均值回歸 | VWAP Reversion | 價格偏離度與均值引力 |

### 2. 門檻自動建議系統

為了降低人工調參的門檻，我們開發了基於歷史數據的門檻建議引擎。該引擎會分析近 30 輪的掃描結果，當偵測到特定阻擋原因的比例過高時，系統會自動生成具體的參數調整建議。

目前支援的建議規則包括：
- **時效性放寬**：當「訊號過舊」成為主要阻擋原因時，建議放寬 `max_signal_age_bars`。
- **樣本數降階**：若「歷史樣本不足」導致阻擋率大於 70%，建議降低 `min_filtered_trades`。
- **評分門檻微調**：當「最新評分不足」頻繁發生，建議適度降低 `min_signal_score`。
- **高週期過濾警示**：若「1D EMA200 方向不符」持續阻擋，系統會提示市場可能處於盤整期，建議暫時放寬高週期同向要求。

### 3. 診斷趨勢視覺化

在前端介面中，我們為每個策略新增了「趨勢序列迷你圖」。這項功能透過不同顏色的色塊，直觀地呈現了策略在最近 30 輪掃描中的狀態變化。

- **綠色**：成功發送信號
- **藍色**：重複信號略過
- **橙色**：條件不符被阻擋
- **灰色**：無信號待機
- **紅色**：執行錯誤

透過這項視覺化工具，交易者可以一眼看出策略近期的活躍頻率與穩定性，無需逐行閱讀日誌。

### 4. 介面文案與 API 優化

在前端細節方面，我們全面盤點了 Dashboard 的文案，將殘留的簡體中文（如「观望」、「确认」等）統一修正為繁體中文，確保介面的專業度與一致性。

同時，後端新增了 `/api/diagnostics-summary` 端點。這個輕量級的 API 專門用於傳輸家族聚合數據、門檻建議與趨勢序列，大幅降低了前端輪詢時的資料傳輸量，提升了系統的整體效能。

## 部署與驗證結果

所有改良項目已透過 `pnpm run build` 重新編譯，並使用 `pm2` 重啟了 Dashboard Server 與 Live Worker。

經由瀏覽器實際驗證，新的「策略家族聚合診斷」與「門檻調整建議」面板已成功渲染於實盤資料明細下方。趨勢序列迷你圖也能正確反映各策略的歷史狀態。目前系統運行穩定，Live Worker 升級至 v4.3 版本，持續以 2 分鐘的間隔進行市場掃描與策略運算。
